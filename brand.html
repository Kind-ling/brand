01 — Logo System
The mark
The Kindling logo pairs a geometric mark (three ascending bars evoking stacked kindling / signal strength / upward trajectory) with the wordmark in Anybody. The mark works alone at small sizes; the full lockup is used in contexts with sufficient space.
Dark background (primary)
Light background
Ember background (accent use)
Mark only (favicons, small)
Do

Use the full lockup on dark backgrounds wherever possible. Maintain generous clear space (minimum 1× mark height on all sides). Use the mark alone only at sizes below 120px wide.

Don't

Don't rotate, stretch, recolor individual bars, add effects/shadows, or place on busy backgrounds without a scrim. Don't use the ember-background variant as the default.

03 — Typography
Three voices
Anybody for display (assertive, geometric, modern). Source Serif 4 for body (readable, warm, trustworthy). IBM Plex Mono for code and data (precise, utilitarian, honest).
Display — Anybody 700
Agent Economy Infrastructure
Usage: headings, hero text, product names, section titles
Weights: 600 (subheads), 700 (primary), 800 (impact)
Body — Source Serif 4
Kindling is trustworthy not because PUC promises neutrality, but because the system minimizes discretionary power: methodology is public, rankings are reproducible, first-party inventory is labeled, routing decisions are exportable and explainable.
Usage: body text, descriptions, long-form documentation
Weights: 300 (body), 400 (emphasis), 600 (strong)
Mono — IBM Plex Mono
npx @kindling/igniter init
referral_split_pct: 15
is_first_party: true
methodology_version: "1.2.0"
Usage: code, data, labels, badges, metadata, terminal output
Weights: 300 (secondary), 400 (default), 500 (labels), 600 (emphasis)

06 — Voice & Messaging
How Kindling speaks
Infrastructure, not hype. Precision, not jargon. Honest about trade-offs. Respectful of the reader's intelligence.
Do

"Kindling Verifier scores services across three dimensions: demand, reliability, and outcome. Methodology is published and reproducible."

Don't

"Our revolutionary AI-powered reputation engine leverages cutting-edge on-chain analytics to deliver unparalleled trust scores."

Do

"PUC operates services that participate in Kindling. First-party services are labeled and receive no scoring advantages."

Don't

"Built by the community, for the community!" (when it's built by PUC)

Do

"IntentForge improved tool selection accuracy by 27% in our benchmarks. Methodology and raw data: [link]."

Don't

"IntentForge delivers 10x better results than anything else on the market."

Core Messages
Tagline
Open infrastructure for the agent economy.
One-liner
Transparent discovery, reputation, and settlement rails for AI agent services.
Elevator
Kindling is a set of open-source modules that make it easy to build, discover, evaluate, and pay for agent-native services. Each module is independently useful. Together, they create reinforcing loops across MCP, A2A, and x402. Economics are on-chain and auditable. Built by Permanent Upper Class.
